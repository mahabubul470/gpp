// Project status

// Why gpp
Built for how code
is actually written now.
Git was designed for humans making deliberate commits. AI agents don't work that way. Neither should your version control.
01
Continuous Capture
Every file change recorded automatically. Two layers: a high-frequency timeline for safety and a curated history for collaboration. Never lose work between commits again.
02
Graphex Knowledge Layer
A versioned, encrypted knowledge graph inside your repo. Agents query project architecture, conventions, and domain terms locally. Context stays yours — nothing leaks.
03
Agent-Aware Collaboration
First-class agent identity. Sandboxed exploration branches. Try → evaluate → accept workflow instead of commit → revert. Parallel approach testing built in.
04
Semantic Diffing
Structure-aware, not line-aware. A variable rename across 40 files shows as one semantic operation. AI refactors become reviewable in seconds.
05
Zero-Knowledge Sync
CRDT-based peer-to-peer replication. No central server dependency. Agents share graph updates without exposing context windows. Works offline.
06
Compliance Built In
Policies-as-code enforced at the storage layer. Full provenance chain from prompt → agent → changeset → deploy. Audit-ready by default.

✕ Line-based diffs — AI refactors unreadable

✕ Central platform dependency (GitHub outages)
✕ File-path CODEOWNERS — stale within weeks
✕ No cost visibility on AI-generated changes
✕ Conflict markers dumped in files — figure it out

✓ Semantic diffs — renames and moves as one op
✓ Agent identity, trust scores, sandboxing
✓ Graphex — persistent, encrypted project brain
✓ Policy engine blocks secrets at storage layer
✓ P2P CRDT sync — no single point of failure

✓ Token-level cost attribution per changeset
✓ AI-assisted conflict resolution with intent
// Under the hood
How it actually works.
No magic. A Cargo workspace of focused crates, content-addressed storage, an embedded SQLite timeline, and changesets that form a DAG — all pure Rust, single binary, no runtime services.

Object store · gpp-core

  • Content addressing with BLAKE3, base32 ids (52 chars, short = 8)
  • Blob / Tree / Changeset / Intent objects
  • Framed GPP\0 wire format: zstd payload + 4-byte checksum
  • Atomic temp-then-rename writes, idempotent, hash-verified reads

Timeline · gpp-timeline

  • SQLite in WAL mode, schema per docs/DATA_MODEL.md
  • Scanner: mtime/size fast-path, blobs written to the store
  • .gppignore + configured ignore via globset
  • Debounced notify watcher; configurable retention prune

History · gpp-history

  • promote folds unpromoted entries → one Changeset
  • Snapshot Tree + Intent (type, task ref, message)
  • Parent-linked changeset DAG; file-based branch refs + HEAD
  • MessagePack bodies, BLAKE3-addressed, walked for gpp log

Stack & conventions

  • Rust 2024 · thiserror/anyhow · tracing · clap
  • UTC i64 µs timestamps; costs in integer micro-dollars
  • No unwrap() in libs; #![forbid(unsafe_code)]
  • CI gates: cargo fmt · clippy -D warnings · test
// Quickstart
Build from source.
Not on crates.io yet — it's a pre-1.0 project. Clone and build the workspace; you get a single gpp binary.
terminal
$ git clone https://github.com/mahabubul470/gpp
$ cd gpp && cargo build --release
$ ./target/release/gpp init
  Initialized empty gpp repository in .gpp/
$ ./target/release/gpp timeline # start capturing
// Interface
Real commands, today.
Every command below works in the current build (gpp 0.1.0). --json on most for agent/script use. The last block is the roadmap.
Init & capture
$ gpp init --graphex
  Initialized empty gpp repository in .gpp/
$ gpp timeline -n 3 # newest first
  #3 4s ago human:dev1 2 files changed
  #2 3m ago human:dev1 1 file changed
$ gpp timeline watch # debounced live stream
Promote & history
$ gpp promote -m "add retry" --intent feature
  Promoted 3 timeline entries → cs:k4f9b2qa on main
$ gpp log --oneline
  cs:k4f9b2qa add retry
$ gpp status
  On branch: main (cs:k4f9b2qa) · 1 unpromoted
Diff & branches
$ gpp diff # working tree vs HEAD
  --- a/src/lib.rs
  +++ b/src/lib.rs
  - let n = 1;
  + let n = retries;
$ gpp branch create feature/x
  created branch feature/x at cs:k4f9b2qa
